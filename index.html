Área Principal

Comandos de Trabajo

Estos son los comandos esenciales que usarás en tu día a día como estudiante de programación

01

git init

Inicializa un nuevo repositorio local en la carpeta actual. Es el primer paso para empezar a usar Git en cualquier proyecto.

Ejemplo 
# Crear nuevo repositorio
$ git init
Initialized empty Git repository in /mi-proyecto/.git/
Primer paso Fácil
02

git clone

Descarga una copia exacta de un repositorio remoto, incluyendo todo el historial, ramas y archivos. Ideal para empezar a colaborar.

Ejemplo 
# Clonar repositorio remoto
$ git clone https://github.com/usuario/repo.git
Cloning into 'repo'...
remote: Enumerating objects: 100, done.
Receiving objects: 100% (100/100), done.
Descarga Fácil
03

git add

Prepara los cambios (staging) para ser incluidos en el próximo commit. Puedes agregar archivos individuales o todos a la vez con .

Ejemplo 
# Agregar todos los cambios
$ git add .

# Agregar archivo específico
$ git add index.html
Staging Fácil

Nota: Estos tres comandos forman la base de todo flujo de trabajo en Git. Domínalos primero antes de explorar los comandos avanzados.

Básico

git commit

Guarda los cambios preparados en el historial del repositorio con un mensaje descriptivo. Crea un punto de control.

Ejemplo 
# Commit con mensaje
$ git commit -m "Agrega navbar responsive"
[main a1b2c3d] Agrega navbar responsive
 2 files changed, 45 insertions(+)
Snapshot Fácil
Básico

git status

Muestra el estado actual del repositorio: archivos modificados, en staging, sin seguimiento y la rama actual.

Ejemplo 
# Ver estado del repo
$ git status
On branch main
Changes to be committed:
  (use "git restore --staged..." to unstage)
        modified:   style.css
Estado Fácil
Ramas

git branch

Gestiona las ramas del repositorio. Permite listar, crear, renombrar y eliminar ramas para trabajar en paralelo.

Ejemplo 
# Listar todas las ramas
$ git branch -a
* main
  feature/login
  remotes/origin/main

# Crear nueva rama
$ git branch feature/header
Ramificación Medio
Ramas

git checkout

Cambia entre ramas o restaura archivos. Permite navegar por el historial y crear ramas nuevas al cambiar.

Ejemplo 
# Cambiar a rama existente
$ git checkout main

# Crear y cambiar a nueva rama
$ git checkout -b feature/navbar
Switched to a new branch 'feature/navbar'
Navegación Medio
Ramas

git merge

Combina el historial de dos ramas. Integra los cambios de una rama en la rama actual donde te encuentras.

Ejemplo 
# Fusionar rama feature en main
$ git checkout main
$ git merge feature/login
Merge made by the 'ort' strategy.
 style.css | 10 ++++++++++
 1 file changed, 10 insertions(+)
Fusión Medio
Remoto

git push

Sube los commits locales al repositorio remoto. Comparte tu trabajo con el equipo en GitHub, GitLab, etc.

Ejemplo 
# Subir cambios a remoto
$ git push origin main
Enumerating objects: 5, done.
Writing objects: 100% (5/5), 450 bytes, done.
To github.com:usuario/repo.git
   a1b2c3d..e4f5g6h  main -> main
Subir Medio
Remoto

git pull

Descarga cambios del remoto y los fusiona automáticamente. Equivalente a git fetch + git merge.

Ejemplo 
# Actualizar desde remoto
$ git pull origin main
remote: Enumerating objects: 3, done.
From github.com:usuario/repo
 * branch            main     -> FETCH_HEAD
Updating a1b2c3d..e4f5g6h
Actualizar Medio
Avanzado

git log

Muestra el historial de commits con información detallada. Incluye hash, autor, fecha y mensaje de cada commit.

Ejemplo 
# Historial gráfico compacto
$ git log --oneline --graph --all
* a1b2c3d (HEAD -> main) Agrega footer
* b2c3d4e Corrige responsive
* c3d4e5f Primer commit
Historial Medio
Avanzado

git stash

Guarda cambios temporalmente sin hacer commit. Útil para cambiar de rama rápidamente sin perder trabajo.

Ejemplo 
# Guardar cambios temporalmente
$ git stash push -m "WIP: navbar"
Saved working directory and index state
On main: WIP: navbar

# Recuperar cambios
$ git stash pop
Temporal Avanzado
Avanzado

git rebase

Reaplica commits sobre otra rama base. Crea un historial lineal y limpio, ideal para proyectos profesionales.

Ejemplo 
# Rebase interactivo
$ git checkout feature
$ git rebase -i main
Successfully rebased and updated feature.
Reorganizar Avanzado
Avanzado

git reset

Deshace cambios moviendo el HEAD. --soft conserva staging, --hard elimina todo. ¡Usar con cuidado!

Ejemplo 
# Deshacer último commit (conserva cambios)
$ git reset --soft HEAD~1

# ⚠️ Eliminar commit y cambios
$ git reset --hard HEAD~1
Deshacer Avanzado
